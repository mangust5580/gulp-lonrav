Стартовый скелет проекта

Эта страница — краткая документация по ожидаемой структуре src/. Замените её своим HTML.

Структура Команды Фичи Docs

Минимальная структура src/

src/
  pages/
    index.html            # entry шаблонов (engine=html по умолчанию)
  styles/
    main.scss             # entry стилей (engine=scss по умолчанию)
  scripts/
    main.js               # entry скриптов
  assets/
    favicons/
      favicon.svg         # source для генерации favicon (по умолчанию обязателен в build)

Папки вывода: dist/ (dev) и public/ (build/preview).

Команды

  • gulp — dev (dist + server + watch)
  • gulp build — production сборка в public/
  • gulp build:fast — ускоренная production сборка (без тяжёлых оптимизаций)
  • gulp preview — build в public/ и запуск сервера

Полная документация находится в README.md в корне репозитория.

Ассеты и фичи

Флаги фич лежат в config/features.js. Большинство модулей опциональны. Для некоторых действует строгая политика: если фича выключена, но входные файлы существуют — в dev будет предупреждение, а в build — ошибка.

Изображения

Растровые картинки и одиночные SVG:

src/assets/images/...

SVG sprite

Иконки для спрайта (один SVG = одна иконка):

src/assets/icons/*.svg

Если спрайт не нужен — отключите в config/features.js.

Static

Файлы копируются как есть в корень output:

src/static/**

Шрифты

Опциональная директория:

src/assets/fonts/**

Shared

Папка для общих фрагментов/partials (используется шаблонизаторами и попадает в watch):

src/shared/**

Если используете чистый HTML — можно игнорировать. Для Nunjucks/EJS/Handlebars — это удобное место для includes.

basePath и linksMode

Для деплоя в подкаталог (например, /my-site/) настройте basePath и linksMode в config/site.js и конфиге валидаторов. Сборка содержит stage-aware валидацию ассетов/ссылок.
